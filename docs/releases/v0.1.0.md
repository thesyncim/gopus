# gopus v0.1.0

Draft status: this is not a published release until the `v0.1.0` tag and the
matching GitHub Release both exist.

## Highlights

- Pure Go Opus encode/decode with no `cgo` or external C toolchain
- caller-owned zero-allocation `Encode` / `Decode` hot path
- Ogg Opus container support via `container/ogg`
- multistream encode/decode support
- libopus-relative parity, provenance, and release-evidence workflow

## Stable Core

The intended stable core for `v0.1.0` is:
- `Encoder`
- `Decoder`
- `MultistreamEncoder`
- `MultistreamDecoder`
- `container/ogg`
- caller-owned `Encode` / `Decode` hot-path APIs

## Supported Default-Build Surface

The default build is the supported public contract for `v0.1.0`.

## Optional Extension Contract

Supported optional controls in the default build:
- `SetDNNBlob(...)` on `Encoder`, `Decoder`, `MultistreamEncoder`, and `MultistreamDecoder`

`make test-dnn-blob-parity` validates the default `SetDNNBlob(...)` control surface against pinned libopus USE_WEIGHTS_FILE model blobs and fails on skipped helper coverage. Decoder-side `SetDNNBlob(...)` currently covers loader-derived validation and retained control state. In DRED-tagged or quarantine builds, `SetDNNBlob(...)` may also bind DRED/OSCE-capable model families on the control path, but normal encode/decode runtime work remains dormant until a DRED duration, payload, recovery path, or explicit quarantine OSCE control is armed. Model-only public caller-buffer encode/decode paths stay zero-allocation and skip unarmed optional helper work. OSCE BWE/LACE runtime behavior is quarantined and parity-gated by numerical libopus-backed forward-pass comparators plus runtime smoke tests; ordinary non-DRED model-backed PLC remains incomplete in `v0.1.0`.

Unsupported or tag-gated in the default build:
- QEXT controls are supported only when built with `-tags gopus_qext`, which exposes `SetQEXT(...)` / `QEXT()` on `Encoder` and `MultistreamEncoder`; `make test-qext-parity` enforces the supported QEXT packet-extension seams against a separate pinned libopus build configured with `--enable-qext` and fails on skipped libopus-helper coverage, while default builds keep QEXT controls absent and do not scan, arm, or encode QEXT packet extensions in public hot paths
- DRED control and standalone surfaces are supported only when built with `-tags gopus_dred`, which exposes `SetDREDDuration(...)` / `DREDDuration()` and standalone `DREDDecoder` / `DRED` helpers; the internal encoder DRED runtime, top-level decoder DRED internals, and multistream decoder DRED cache/runtime helpers are build-tag split out of default `.`, `./encoder`, and `./multistream` builds; standalone DRED wrapper lifecycle/no-allocation, libopus parse/decode/process metadata checks, real-packet standalone process state/feature parity, standalone recovery scheduling parity, top-level decoder dormancy/cache lifecycle coverage, decoder cached recovery bookkeeping parity, stereo cached recovery lifecycle/cursor seams, SILK wideband 20/40/60 ms mono and 20 ms stereo carried-payload/packet-envelope seams, Hybrid fullband 20/40 ms mono/stereo carried-payload/packet-envelope seams, and final/non-final uncoupled mono, final/non-final single-coupled stereo, and final/non-final non-leading second-coupled multistream CELT/Hybrid/SILK DRED consumers are enforced under `make test-dred-tag`; quarantine API exposure, standalone/control smoke, top-level decoder dormancy/cache lifecycle coverage, cached bookkeeping, multistream DRED consumer, and dormant-runtime checks are pinned by `make test-unsupported-controls-tag`; those encoder seams plus parser availability, internal converter/payload/basic-analysis coverage, real-model PitchDNN and RDOVAE encoder oracles, the libopus DRED latent trace guard, the conceal-analysis oracle, 48 kHz bootstrap coverage, the required mono decoder explicit/live numerical matrix, selected 16 kHz Hybrid mono live-sequence seams, CELT/Hybrid stereo cached/live first/second-loss and next-packet handoff matrices, selected 16 kHz CELT/Hybrid stereo explicit first-loss probes, explicit first-loss and recovery lifecycle/cursor seams, 48 kHz / 16 kHz SILK WB explicit mono first-loss seams, the 48 kHz SILK WB explicit stereo first-loss seam, stereo cached recovery lifecycle/cursor seams, and focused multistream CELT/Hybrid/SILK DRED consumers are mirrored by `make test-unsupported-controls-parity`; required DRED parity gates fail on skipped libopus-helper tests, broader Hybrid/SILK primary-frame byte exactness remains outside the support claim unless named in the byte-exact test matrix, and broader SILK stereo plus multistream packet/mode coverage remains seam-specific
- Supported tags may be combined: `-tags "gopus_dred gopus_qext"` reports DRED and QEXT support, while `-tags "gopus_unsupported_controls gopus_qext"` reports QEXT support and leaves DRED/OSCE BWE unsupported despite exposing quarantine controls
- `SetOSCEBWE(...)` / `OSCEBWE()` are absent unless built with `-tags gopus_unsupported_controls`
- The `gopus_unsupported_controls` build remains a parity/quarantine umbrella and does not itself make `SupportsOptionalExtension(...)` report feature support

The `gopus_unsupported_controls` build may expose DRED controls and standalone
helpers for quarantine parity, but release support still comes only from
`gopus_dred`.

Low-level implementation packages remain pre-v1 and may change more freely than the top-level `gopus` facade plus `container/ogg`.

## Support Matrix

- Go: `1.25+`
- Tested in CI: Linux, macOS, Windows
- Optimized kernels: `amd64` and `arm64`
- Pure-Go fallbacks: available on other architectures

## Verification Evidence

This release is expected to ship with evidence generated from:
- `go test ./...`
- `make test-doc-contract`
- `make lint`
- `make test-consumer-smoke`
- `make verify-production`
- `make verify-production-exhaustive`
- `make release-evidence`

The GitHub Release must attach:
- `release-evidence-v0.1.0.md`, the readable evidence summary
- `release-evidence-v0.1.0.tar.gz`, the evidence bundle with command logs and inventories

The attached evidence must include:
- commit SHA
- Go version
- OS/platform
- pinned libopus version and tarball SHA256
- command manifest and pass/fail summaries
- benchmark guardrail result
- fuzz/safety summary
- parity summary
- consumer-smoke result
- `TestEncoderComplianceSummary`
- `make test-exactness`
- hot-path allocation guard results
- race test results

## Known Limits

- This is the first tagged public release and remains pre-v1.
- Some optional libopus build-time extensions remain intentionally unsupported in the default build.
- Low-level/internal-facing packages may continue to change faster than the stable core.
